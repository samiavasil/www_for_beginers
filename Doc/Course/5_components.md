# 05. Основи на компоненти (React/Vue/LWC)
# Какво са компонентните рамки?
Компонентните рамки като React, Vue и LWC са инструменти, които помагат да създаваш уеб приложения по-структурирано и удобно. Те използват JavaScript, но добавят нови възможности:
- Можеш да разделиш приложението на малки части (компоненти)
- Всеки компонент има собствен HTML, CSS и JS
- Компонентите могат да се използват многократно и да се "импортват" в други части на приложението

## Каква е разликата със стандартния JS?
Стандартният JS ти позволява да пишеш код директно в HTML или във външен файл. Компонентните рамки ти дават структура и автоматизация – например можеш да създадеш компонент за бутон и да го използваш навсякъде, без да копираш кода.

## Какво е React?
React е популярна рамка за компоненти, създадена от Facebook. В React всеки компонент е JS функция или клас, който връща HTML (JSX). Компонентите се "импортват" и използват като тагове.

## Какво е Vue?
Vue е друга рамка за компоненти, която е лесна за начинаещи. В Vue компонентите се описват с шаблон (HTML), скрипт (JS) и стил (CSS) в един файл.

## Какво е LWC?
LWC (Lightning Web Components) е стандарт за компоненти, използван в Salesforce. Подобно на React и Vue, позволява да създаваш многократно използваеми части, но е насочен към корпоративни приложения.

## Пример за импорт на компонент (React)
```js
import UserCard from './UserCard';

function App() {
  return <UserCard name="Иван" />;
}
```
В този пример UserCard е компонент, който се използва като таг в приложението.

## За начинаещи
Преди да преминеш към React, Vue или LWC, е важно да научиш основите на стандартния JS и как се "симулират" компоненти с функции и HTML. После можеш да разгледаш рамките и да видиш как те улесняват работата.

## Какво са компонентите?
Компонентите са малки, самостоятелни части от уеб приложението, които могат да се използват многократно. Те помагат да разделиш кода на логични блокове – например меню, бутон, форма.

## Защо са важни?
- Улесняват повторното използване на код
- Правят структурата на проекта по-ясна
- Позволяват по-лесна поддръжка и развитие

## Пример с HTML (нюби вариант)
В HTML можеш да "симулираш" компонент чрез отделен `<div>` или `<section>`:
```html
<section id="userCard">
  <h2>Потребител</h2>
  <p>Име: Иван</p>
  <button>Покажи профил</button>
</section>
```

## Пример с JavaScript (нюби вариант)
Можеш да създадеш функция, която "генерира" HTML за компонент:
```html
<div id="app"></div>
<script>
function createUserCard(name) {
  return `<section class="userCard">
    <h2>Потребител</h2>
    <p>Име: ${name}</p>
    <button>Покажи профил</button>
  </section>`;
}

document.getElementById('app').innerHTML = createUserCard('Иван');
</script>
```

## Какво е LWC?
LWC (Lightning Web Components) е модерен стандарт за компоненти, използван в Salesforce. Подобно на React и Vue, позволява да създаваш многократно използваеми части.

## Нови тагове и концепции
- `<section>`: използва се за отделни части ("компоненти")
- JavaScript функции: могат да "генерират" HTML
- Атрибути: id, class – помагат да идентифицираш и стилизираш компонентите

## Задача
Създай HTML файл с "компонент" – например карта на потребител, която се показва чрез JavaScript функция.
